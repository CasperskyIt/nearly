## Spacing Scale

All values are multiples of 4. Tokens map to the values already used in `home.component.scss` and `styles.scss`. Do not introduce values outside this scale.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, `.header-left gap` |
| sm | 8px | Card margin-bottom, filter chip gaps, zoom control gaps |
| md | 16px | Default element padding (header `padding: 0 16px`, sidebar header padding) |
| lg | 24px | Section padding, place detail panel (`padding: 20px 24px`) |
| xl | 32px | Welcome CTA padding, major content gaps |
| 2xl | 48px | Loader container padding (40px rounds to this scale) |
| 3xl | 64px | Fixed header height |

Exceptions:
- Header height is fixed at 64px — this aligns with `--z-header` / `.header` height and must not change during refactor
- Filters bar height is fixed at 56px — do not alter during MapComponent extraction
- Sidebar width is fixed at 340px — referenced in `.map-container` and `.sidebar-toggle` positioning
- Zoom button touch target is 48px x 48px — intentional above-scale value for tap target compliance

## Component Extraction: Visual Constraints

These constraints apply specifically to the HomeComponent decomposition (FOUND-02). They prevent visual regressions during refactor.

### MapComponent

- Must occupy the same fixed region: `top: 120px; left: 340px; right: 0; bottom: 0`
- When sidebar is collapsed: `left: 0` (`full-width` modifier class)
- Leaflet map must render inside a div with `id="map"` or equivalent — Leaflet requires a named DOM container
- Zoom controls (48px x 48px buttons, `bottom: 24px; right: 24px`) move into `MapComponent` template
- `leaflet-control-zoom` is hidden via global CSS — this must remain in `styles.scss`, not scoped to `MapComponent`

### PlacesListComponent

After conversion to a presentational component (`input<Place[]>([])`):

- Loading state: show the existing `.loader-container` markup with `.spinner` animation when `places` input is empty and parent signals loading
- Empty state: see Copywriting Contract below — no places visible must show a human message, not an empty list
- Place cards preserve existing `.place-card` layout (64px icon, flex row, `--radius-md` corners)

### HomeComponent (orchestrator, under 100 lines)

- Must preserve `ThemeService` injection — logo and color overrides depend on it
- The `.app-container` wrapper with potential `.dark-mode` modifier must remain on the root element

## Copywriting Contract

Phase 1 removes mock data. After removal, the places list will be empty until a real Overpass API response arrives. These states are now real user-facing moments that require defined copy.

| Element | Copy | Notes |
|---------|------|-------|
| Loading state heading | "Finding places near you" | Shown while OSM request is in flight |
| Loading state body | (none — use spinner only) | Spinner communicates progress; no extra text needed |
| Empty state heading | "No places found nearby" | Shown when Overpass returns 0 results (valid API response) |
| Empty state body | "Try a different category or move the map to a new area." | Gives the user a clear next action |
| OSM error state | "Could not load places. Check your connection and try again." | Shown on Overpass API failure — replaces `console.error` visible only in devtools |
| Primary CTA (welcome screen) | "Find places near me" | Existing Dogly string — do not change (lives in `theme.config.ts strings.welcomeButton`) |
| Destructive confirmation | Not applicable | No destructive user actions exist in Phase 1 |

> The OSM error copy is new — it must be surfaced in the UI, not swallowed by `LoggerService`.
> `LoggerService` suppresses console output in production; a visible error state must still render
> so the user knows the map is not broken.

## Interaction Contract

These are the only interactions that exist in Phase 1 (no new ones introduced):

| Interaction | Trigger | Response | Timing |
|-------------|---------|----------|--------|
| Category filter select | Tap filter chip | Chip enters `.active` state; map reloads with category filter | `--transition-fast` (150ms) |
| Place card select | Tap place card | Card gets `selected` border; map pans to marker; detail panel slides up | `slideUp` animation (`--transition-normal` 250ms) |
| Detail panel close | Tap close button | Panel dismisses; card deselects | Immediate |
| Sidebar collapse/expand | Tap sidebar toggle | Map container transitions `left` value | `--transition-normal` (250ms) |
| Welcome dismiss | Tap anywhere on welcome overlay | Overlay fades out | `--transition-normal` (250ms) |

Hover states use `--transition-fast` (150ms) throughout. Do not change transition values during refactor.

## LoggerService: No Visual Surface

`LoggerService` is a no-op in production — it has no UI component. The only visual impact:

- Remove: `console.log('Toggle menu')` and `console.error(...)` calls throughout components
- Add: Where errors were previously logged and silently swallowed, surface the OSM error state (see Copywriting Contract above)
- The error boundary is in `HomeComponent` (or `MapComponent` after extraction) — catch the Overpass error signal and show the error copy in the places sidebar area
