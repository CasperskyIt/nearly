**Component logic:**
- `data = inject<{ dog: Dog }>(MAT_DIALOG_DATA)`
- `dialogRef = inject(MatDialogRef<DogEditDialogComponent>)`
- `dogService = inject(DogService)`
- `snackBar = inject(MatSnackBar)`
- `saving = signal(false)`
- `avatarPreview = signal<string | null>(null)` — local preview URL from FileReader
- `selectedFile = signal<File | null>(null)` — file to upload on save

- Form initialized with current dog values:
  - `name: FormControl<string>(data.dog.name, Validators.required)`
  - `breed: FormControl<string>(data.dog.breed ?? '')`
  - `dateOfBirth: FormControl<Date | null>(data.dog.date_of_birth ? new Date(data.dog.date_of_birth) : null)`

- `onFileSelected(event: Event)`:
  1. Get file from `(event.target as HTMLInputElement).files?.[0]`
  2. Validate size: if `file.size > 5 * 1024 * 1024` (5MB per D-12), show snackbar "File too large. Maximum 5MB." and return
  3. Validate type: if not `image/jpeg`, `image/png`, or `image/webp`, show snackbar "Only JPEG, PNG, and WebP images are allowed." and return
  4. Set `selectedFile(file)`
  5. Read file preview: `const reader = new FileReader(); reader.onload = (e) => this.avatarPreview.set(e.target?.result as string); reader.readAsDataURL(file);`

- `async onSave()`:
  1. If form invalid, mark touched, return
  2. `saving.set(true)`
  3. If `selectedFile()` is not null, upload avatar first: `const url = await this.dogService.uploadAvatar(data.dog.id, selectedFile()!)`. If null, show error snackbar, set saving false, return.
  4. Build UpdateDogRequest: `{ name: form.value.name, breed: form.value.breed || null, date_of_birth: form.value.dateOfBirth ? form.value.dateOfBirth.toISOString().split('T')[0] : null }`
  5. Call `dogService.updateDog(data.dog.id, request)`
  6. If success: show snackbar "Dog updated" (per D-32), close dialog with updated dog object: `dialogRef.close(result)`
  7. If failure: show snackbar "Failed to update dog" (per D-31)
  8. `saving.set(false)` in finally
