    Create `frontend/src/app/pages/auth-callback/auth-callback.component.ts` (per D-23 — must not redirect before Supabase parses URL):
    - Standalone component
    - Inject: `SupabaseService`, `Router`, `ActivatedRoute`
    - In `ngOnInit()`:
      1. Subscribe to `supabase.supabase.auth.onAuthStateChange((event, session) => {...})`
      2. On `event === 'SIGNED_IN' && session`:
         - Unsubscribe the listener (call `subscription.unsubscribe()`)
         - Read `returnUrl` from `this.route.snapshot.queryParamMap.get('returnUrl')`
         - Sanitise: if `returnUrl` is null, empty, or contains `/login`, set to `null`
         - Navigate: `this.router.navigateByUrl(returnUrl ?? '/dogly')` (per D-05)
      3. On `event === 'INITIAL_SESSION' && !session` (per D-14 — no OAuth params):
         - Unsubscribe
         - `this.router.navigateByUrl('/dogly/login')` (silent redirect, no error shown)
      4. CRITICAL: Do NOT navigate in `ngOnInit` before the auth state change fires. The component just renders a loading message while the PKCE exchange happens.
    - Store the subscription return value: `const { data: { subscription } } = this.supabase.supabase.auth.onAuthStateChange(...)`
